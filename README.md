# ⚡ AutoFlow — 可视化自动化管理平台

> 基于 **CustomTkinter** + **PyAutoGUI** 构建的桌面端 RPA 工具。
> 通过可视化界面配置，将重复性的鼠标点击、表单填写、文件上传等操作封装成自动化任务，并对外暴露 HTTP API 接口，支持程序化调用。

## 功能概览

| 功能 | 说明 |
|------|------|
| 🖥 可视化配置 | 通过界面添加组件，设置坐标和参数，无需写代码 |
| 📋 任务管理 | 列表管理所有自动化任务，支持新建、编辑、删除、发布 |
| ▶ 本地执行 | 界面内填写参数后一键执行，实时查看日志 |
| 🔌 HTTP API | 启动内置服务器，通过 REST 接口远程调用任务 |
| 💾 本地持久化 | 所有任务自动保存到本地 JSON 文件，重启不丢失 |

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

`requirements.txt` 内容：
```
customtkinter>=5.2.0
pyautogui>=0.9.54
Pillow>=10.0.0
```

### 2. 启动应用

```bash
python main.py
```

### 3. 创建第一个任务

1. 点击顶部 **「⚙ 配置编辑」** 选项卡（或管理中心的「＋ 新建自动化」）
2. 填写任务名称
3. 从左侧面板依次添加组件，为每个组件设置名称、坐标、等待时长
4. 点击 **「💾 保存」**
5. 在管理中心找到该任务，点击 **「🔗 发布」** 即可通过 API 调用

## 使用指南

### 管理中心

管理中心以表格形式展示所有自动化任务。

| 操作 | 方式 |
|------|------|
| 进入编辑 | 双击行，或点击 ✏ 编辑按钮 |
| 发布任务 | 点击 🔗 发布，生成 API 接口 |
| 删除任务 | 点击 🗑 删除（需二次确认） |
| 搜索任务 | 顶部搜索框实时过滤 |

**任务状态说明：**
- `○ 草稿` — 仅可在界面内手动执行，不可通过 API 调用
- `● 已发布` — 可通过 HTTP API 远程调用

---

### 配置编辑

配置编辑页面分为左右两栏：

**左侧 — 属性面板**
- 填写任务名称
- 点击对应按钮添加组件（每次点击弹出添加弹窗）

**右侧 — 画布**
- 已添加的组件以卡片形式从上到下排列，**执行顺序与排列顺序一致**
- 每张卡片内可直接修改名称、坐标、等待时长
- **右键任意组件** → 弹出菜单，可上移 / 下移 / 删除

---

### 执行任务

在管理中心点击 ✏ 编辑，进入配置页后点击执行；或在执行弹窗中操作。

**执行弹窗说明：**

| 区域 | 说明 |
|------|------|
| 左侧参数区 | 为输入组件和上传组件填写运行时的值 |
| 右侧日志区 | 实时显示每一步的执行过程 |
| 3 秒倒计时 | 点击执行后有 3 秒缓冲，用于切换到目标软件窗口 |
| ⏹ 停止按钮 | 随时中断执行 |

> **紧急停止**：将鼠标快速移动到屏幕**左上角**（坐标 0,0 附近）可触发 PyAutoGUI FailSafe 机制立即停止。

---

### API 服务

点击标题栏右侧的 **「启动 API」** 按钮，内置 HTTP 服务器将在本地 `8765` 端口启动。

启动成功后标题栏显示 `● http://127.0.0.1:8765`，同时「接口文档」按钮变为可用，点击可查看完整接口说明。

## 组件类型说明

AutoFlow 支持 **5 种组件类型**，执行时按从上到下的顺序依次运行：

---

### 🖥 对象组件（橙色）

**用途**：点击任务栏或桌面图标，打开并激活目标软件。通常放在所有组件的**第一位**。

| 字段 | 说明 |
|------|------|
| 对象名称 | 软件名称，如「微信」「Chrome」 |
| 图标 X / Y | 任务栏或桌面图标的屏幕坐标 |
| 启动等待（秒） | 点击后等待软件加载的时间（建议 2 ~ 5 秒） |

**执行动作**：移动鼠标 → 点击图标 → 等待 N 秒

---

### 📝 输入组件（绿色）

**用途**：定位到表单输入框，清空原有内容后填写新内容。

| 字段 | 说明 |
|------|------|
| 组件名称 | 字段名，如「用户名」「密码」「手机号」 |
| X / Y | 输入框的屏幕坐标 |
| 等待时长（秒） | 输入完成后等待的时间（通常设 0） |

**执行动作**：移动鼠标 → 点击输入框 → `Ctrl+A` 全选 → 逐字输入内容

> 调用 API 时通过 `params` 字段传入要输入的文本。

---

### 📎 上传组件（蓝色）

**用途**：点击文件上传按钮，在弹出的文件对话框中自动填入文件路径并确认。

| 字段 | 说明 |
|------|------|
| 组件名称 | 如「头像」「附件」「证件照」 |
| X / Y | 上传按钮的屏幕坐标 |
| 等待时长（秒） | 上传完成后等待的时间 |

**执行动作**：移动鼠标 → 点击上传按钮 → 等待文件对话框 → 输入文件路径 → 按回车确认

> 调用 API 时通过 `params` 传入文件的完整路径，如 `C:\Users\xxx\photo.jpg`。

---

### 🖱 按钮组件（红色）

**用途**：点击指定坐标的按钮，如「提交」「确认」「搜索」「登录」等。

| 字段 | 说明 |
|------|------|
| 组件名称 | 如「提交」「确认」「登录」 |
| X / Y | 按钮的屏幕坐标 |
| 等待时长（秒） | 点击后等待的时间（若按钮触发页面跳转，建议设 1 ~ 3 秒） |

**执行动作**：移动鼠标 → 点击

---

### ↵ 回车组件（紫色）

**用途**：移动到指定坐标后按回车键，适用于搜索框确认、下拉选项确认等场景。

| 字段 | 说明 |
|------|------|
| 组件名称 | 如「确认搜索」「选择选项」 |
| X / Y | 目标坐标（先点击再按回车） |
| 等待时长（秒） | 按回车后等待的时间（默认 0） |

**执行动作**：移动鼠标 → 点击 → 按 `Enter` 键

## HTTP API 文档

### 基础信息

```
基础地址：http://127.0.0.1:8765
数据格式：application/json（UTF-8）
跨域：已支持（CORS）
```

---

### GET `/api/ping`

测试服务是否正常运行。

```bash
curl http://127.0.0.1:8765/api/ping
```

```json
{
  "success": true,
  "message": "AutoFlow API 运行中",
  "port": 8765
}
```

---

### GET `/api/tasks`

获取所有任务列表。

```bash
curl http://127.0.0.1:8765/api/tasks
```

```json
{
  "success": true,
  "tasks": [
    {
      "id": "demo01",
      "name": "登录表单自动填写",
      "published": true,
      "component_count": 3,
      "updated_at": "2025-03-30 09:21"
    }
  ]
}
```

---

### GET `/api/tasks/{task_id}`

获取单个任务详情。

```bash
curl http://127.0.0.1:8765/api/tasks/demo01
```

---

### POST `/api/run`

**执行指定任务**（同步，执行完成后返回）。

**请求体：**
```json
{
  "task_id": "任务ID",
  "params": {
    "组件名称": "对应的值"
  }
}
```

**params 填写规则：**

| 组件类型 | 是否需要传参 | 传参格式 |
|---------|------------|---------|
| 📝 输入组件 | ✅ 需要 | `"组件名": "文本内容"` |
| 📎 上传组件 | ✅ 需要 | `"组件名": "C:/path/file.jpg"` |
| 🖱 按钮组件 | ❌ 无需 | 自动点击 |
| ↵ 回车组件 | ❌ 无需 | 自动按回车 |
| 🖥 对象组件 | ❌ 无需 | 自动点击图标 |

**示例（curl）：**
```bash
curl -X POST http://127.0.0.1:8765/api/run \
  -H "Content-Type: application/json" \
  -d '{"task_id":"demo01","params":{"用户名":"admin","密码":"123456"}}'
```

**示例（Python）：**
```python
import requests

res = requests.post("http://127.0.0.1:8765/api/run", json={
    "task_id": "demo01",
    "params": {
        "用户名": "admin",
        "密码": "123456"
    }
})
print(res.json())
```

**示例（JavaScript）：**
```javascript
const res = await fetch("http://127.0.0.1:8765/api/run", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    task_id: "demo01",
    params: { "用户名": "admin", "密码": "123456" }
  })
});
console.log(await res.json());
```

**成功响应：**
```json
{
  "success": true,
  "message": "执行完成 ✓",
  "task_id": "demo01",
  "task_name": "登录表单自动填写",
  "log": [
    "▶ 开始执行：登录表单自动填写",
    "  共 3 个组件",
    "[1/3] 用户名 (input)",
    "  坐标 → (320, 420)",
    "  ✍ 已输入：admin",
    "✓ 所有组件执行完毕"
  ]
}
```

**HTTP 状态码：**

| 状态码 | 含义 |
|--------|------|
| 200 | 执行成功 |
| 400 | 请求参数错误 |
| 403 | 任务未发布 |
| 404 | 任务不存在 |
| 500 | 执行过程出错 |

## 数据存储

所有任务保存在 `data/automations.json`，结构如下：

```json
[
  {
    "id": "demo01",
    "name": "登录表单自动填写",
    "published": false,
    "created_at": "2025-03-30 09:00",
    "updated_at": "2025-03-30 09:21",
    "components": [
      {
        "type": "input",
        "name": "用户名",
        "x": 320,
        "y": 420,
        "value": "",
        "extra_wait": "0"
      },
      {
        "type": "button",
        "name": "登录",
        "x": 320,

  }

```

> 该文件可以直接编辑，修改后重启应用生效。建议定期备份。

## 注意事项

### 获取屏幕坐标

推荐使用以下方式：

- **Windows 画图**：打开画图工具，将鼠标悬停在目标位置，左下角显示像素坐标
- **Python 脚本**：
  ```python
  import pyautogui, time
  print("3 秒后输出鼠标坐标...")
  time.sleep(3)
  print(pyautogui.position())
  ```

### 等待时长建议

| 场景 | 建议时长 |
|------|---------|
| 对象组件（启动软件） | 2 ~ 5 秒 |
| 按钮组件（触发页面跳转） | 1 ~ 3 秒 |
| 按钮组件（弹出对话框） | 0.5 ~ 1 秒 |
| 输入 / 回车组件 | 0 秒（通常不需要） |
| 上传组件（网络上传大文件） | 视文件大小而定 |


### 任务发布

只有在管理中心点击「🔗 发布」的任务才能通过 API 调用。未发布的草稿任务调用 API 会返回 `403` 错误。
